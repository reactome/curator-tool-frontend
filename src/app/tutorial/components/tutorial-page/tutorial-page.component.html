Reactome Curation WebBench — Tutorial

Use the tabs below to explore each area of the application, or launch an interactive guided tour.

rocket_launch Getting Started
login Sign In
  1. Navigate to /login.
  2. Enter your Username and Password and click OK.
  3. If you were redirected from a protected page, you will be returned there automatically.

info Special characters (!, $, etc.) in passwords are not yet supported — use alphanumeric passwords.

home Home Page

After signing in, the home page gives you three launch cards:

  • Schema View — browse classes, list/edit instances, manage staged changes.
  • Event View — combine event tree, pathway diagram, and instance editor.
  • Gene2Path — LLM-assisted pathway annotation for a gene.

The Status Toolbar at the bottom shows staged change counts and quick actions.

 save Staged Changes & Commit Model

The Curator WebBench uses a local staging model. All edits are saved locally in your browser first, then committed to the database when you're ready.

  1. Make edits — instance creates, updates, or deletes are staged.
  2. Review staged changes by clicking any counter in the status toolbar.
  3. Select items in the staged-changes panel and click the upload icon to commit.
  4. Use the reset buttons to undo staged changes without committing.

warning If you see a "too many staged instances" warning, commit your work soon to avoid data loss on browser close.

account_tree Schema View
park Class Tree
  • Click a class name to view its attribute definitions (allowed types, cardinality, origin).
  • Click the [count] badge next to a class to list instances from the database for that class.
  • Click the (count) badge to list only your staged/local instances for that class.
  • Click the + icon to create a new instance of that class.
 list Instance List

Search

  • Type in the search box and press Enter for a quick text/dbId search.
  • Click the filter icon to switch to Advanced Search — add conditions using Contains, Equal, Not Equal, IS NULL, IS NOT NULL.
  • Click the undo icon to remove the last advanced condition.
  • After a search, click the Download icon to export the full result set as CSV (dbId, displayName, schemaClass).

Row Actions

  • Click a row to open the instance editor.
  • Right-click or use the row action menu for: delete, show referrers, copy/clone, compare, show tree.
  • Select rows with the checkbox column for bulk actions.

List-Level Actions

  • Select All on Page / Deselect All
  • Delete Selected
  • Compare Selected (when exactly 2 rows are selected)
  • Batch Edit (see Batch Edit section)
 edit Instance Editor

Toolbar

  • QA icon — run quality checks on this instance.
  • Referrers icon — list all instances that reference this one.
  • Compare icon — diff current state against the database version.
  • Delete icon — stage this instance for deletion.
  • Upload icon — commit this single instance.
  • Bookmark icon — add to your bookmarks strip.

Attribute Types

  • String: edit in the textarea. Press Enter to commit. Press Ctrl+Enter (or ⌘+Enter) to insert a newline.
  • Integer / Float: numeric input only (no spinner arrows). Press Enter to commit.
  • Boolean: radio buttons for True / False.
  • Instance-type: use the action menu (Set, Add, Replace, Delete). Drag a bookmark onto a compatible slot to assign its value.

Multi-Valued Attributes

For multi-valued instance slots you can also reorder values by dragging. Adding a value that already exists will be blocked and you will see a notification — except for stoichiometry attributes (HasComponent, Input, Output, etc.) which allow repeats.

edit_note Batch Edit
  1. Select rows from the instance list (optional — if none selected, batch edit applies to the full list scope).
  2. Click the Batch Edit icon in the search bar.
  3. Choose an attribute and an action.
  4. Fill in the value and confirm.

account_tree Layout
  • Left panel: species filter + event tree (can switch to staged-changes panel with the status toggle).
  • Upper right: pathway diagram.
  • Lower right: instance editor (same editing tools as Schema View).
  • Right edge: bookmarks strip (expandable/collapsible, draggable).
  • Bottom: status toolbar.

All panels are resizable by dragging their dividers.

 filter_list Event Tree & Filter
  • Use the species dropdown to filter the tree by species.
  • Type in the text filter and press Enter to highlight/filter events by name or dbId.
  • Expand/collapse hierarchy by clicking the tree node arrows.
  • Click an event to load it in the instance editor below.
  • Use the tree action icons to: add the event to the diagram, create an empty diagram, or toggle the release flag.
  • Shift+click the release flag to toggle it for the event and all its children.
 schema Pathway Diagram
  • Selecting an event in the tree highlights its objects in the diagram.
  • Clicking a diagram object loads the related instance in the lower editor.
  • Right-click the diagram to open the context menu.

Diagram Context Menu Options

  • Enable / disable diagram editing
  • Enable / disable reaction edge editing
  • Add / remove edge points and flow lines
  • Resize compartments and pathway nodes
  • Insert / delete compartments
  • Align multiple selected nodes vertically or horizontally
  • Toggle color theme
  • Upload diagram changes
  • Reload the current pathway diagram
  • Open or create the related PathwayDiagram instance
  • Navigate to a nested pathway
psychology Gene2Path
science Overview

Gene2Path uses an LLM service to generate pathway annotation support for a gene. Results include annotated pathways, predicted pathways, and protein-protein interaction tables.

info The Gene2Path route does not require authentication and can be shared via URL.

play_arrow Usage
  1. Navigate to Gene2Path from the home page.
  2. Enter a gene symbol in the input field.
  3. Optionally click the gear icon to adjust LLM configuration settings.
  4. Click the Submit (publish) icon.
  5. A progress bar shows while the query runs.
 find_in_page Reviewing Results
  • Use the side navigation menu to jump between generated sections.
  • Annotated Pathways: confirmed pathway associations with evidence.
  • Predicted Pathways: LLM-suggested pathway associations to review.
  • Interaction tables: protein-protein interaction data supporting the annotations.
  • Generated text includes links to Reactome pathways and PubMed where available.

warning Generated content is curation assistance — always review before adding to Reactome.
